# Product Mission

## Problem

AI coding agents lack the product and codebase context needed to work effectively. Without structured context about what they're building, who it's for, and how it should be built, agents produce inconsistent results that don't align with project standards or goals.

## Target Users

Software developers using AI coding assistants (such as Claude Code, Cursor, and similar tools) who want their AI agents to produce higher-quality, more consistent output.

## Solution

Spec-driven development: agents work from structured specs and standards rather than ad-hoc prompts. By establishing a shared context layer — product mission, roadmap, tech stack, and coding standards — developers give AI agents the grounding needed to make better decisions and produce better code.
