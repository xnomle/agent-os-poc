# README Standards

Every repo must have a `README.md` at the root.

## Required Sections

1. **Project title** — what it is in one line
2. **Description** — 2-3 sentences on what it does and why
3. **Prerequisites** — tools, runtimes, accounts needed
4. **Getting started** — clone, install, run locally
5. **Deployment** — how it gets deployed
6. **Architecture** — brief overview of key components

## Rules

- Keep it current — update when infra or setup steps change
- No placeholder sections — if it's not written yet, don't add the heading
- Code blocks for all commands — never describe a command in prose
- Assume the reader has no context on this project
